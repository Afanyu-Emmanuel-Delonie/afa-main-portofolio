# Afanyu Emmanuel — Portfolio & Service Site

A personal portfolio and freelance service site built with React, Vite, and Tailwind CSS v4. Designed to showcase projects, skills, work experience, and services — while also functioning as a client-facing landing page for freelance inquiries.

## Getting Started

### Prerequisites

- Node.js 18+
- npm

### Installation

```bash
git clone https://github.com/your-username/afa-service-site.git
cd afa-service-site
npm install
```

### Development

```bash
npm run dev
```

Opens at `http://localhost:5173`

### Build

```bash
npm run build
```



## Adding a New Project

All project data lives in `src/data/projects.js`. Add a new entry following this shape:

```js
{
  id: 'unique-slug',           // used for modal key
  title: 'Project Name',
  desc: 'Short description shown on the card.',
  stack: ['React', 'Django'],  // tech tags
  live: 'https://...',         // live URL
  github: 'https://...',       // GitHub URL
  color: 'from-blue-500/20 to-purple-500/20', // gradient for card banner
  caseStudy: {
    overview: '...',
    problem: '...',
    solution: '...',
    outcome: '...',
    sections: [
      { heading: 'Section Title', body: 'Section content.' },
    ],
  },
}
```

- The first 3 entries appear as featured highlights on the homepage
- If there are more than 3, a "View All Projects" button appears automatically
- Each card has a "Case Study" button that opens a popup modal — no extra page needed

## Contact Form — Google Sheets Integration

Form submissions are sent to a Google Apps Script Web App and stored in a Google Sheet named `Portfolio-applications` inside a file called `Job-Applications`.

### Sheet Columns

| Timestamp | Full Name | Email | Phone Number | Interest | Message |

### Apps Script

The deployed endpoint:
```
https://script.google.com/macros/s/AKfycbyl2nwFwgvfxs6JxzOAawXI4jlvLPybv4gmSNdVOODmN3fPAlfvIx4eY2sKuUJ3ECis/exec
```

The frontend sends a `POST` request with `application/x-www-form-urlencoded` body using `mode: 'no-cors'` to bypass CORS restrictions. The Apps Script reads fields from `e.parameter`.

### To update the endpoint

Replace the URL in `src/app/components/Contact.jsx` inside `handleSubmit`.

## Scroll Animations

All sections use a lightweight custom scroll reveal system — no external animation library needed.

- `src/hooks/useScrollReveal.js` — `IntersectionObserver` hook that fires once when an element enters the viewport at 12% threshold
- `src/app/components/Reveal.jsx` — wrapper component that accepts `delay` (ms) prop for staggered animations. Applies `opacity` + `translateY` CSS transitions

Usage:
```jsx
<Reveal delay={100}>
  <YourComponent />
</Reveal>
```

## SEO

Configured in `index.html`:

- Title, meta description, keywords, canonical URL
- Open Graph tags (LinkedIn, WhatsApp, Facebook previews)
- Twitter Card (large image)
- JSON-LD structured data (`Person` schema) — readable by Google, ChatGPT, Perplexity, Gemini
- `public/robots.txt` — explicitly allows GPTBot, ChatGPT-User, Google-Extended, PerplexityBot, ClaudeBot
- `public/sitemap.xml` — submitted to Google Search Console

### Before going live, update:

- Replace `https://afanyuemmanuel.dev` with your real domain in `index.html`, `robots.txt`, and `sitemap.xml`
- Add `public/og-image.png` (1200×630px) for social share previews
- Update `sameAs` links in JSON-LD with your real GitHub and LinkedIn URLs
